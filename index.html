SQL Notes

  • 1. View all data
    Used to display every record from a table. 
    SELECT * FROM students;
  • 2. Insert data
    Adds a new row into a table. 
    INSERT INTO students (id, name, age, grade)
VALUES (1, 'Alex', 20, 'A');
  • 3. Update data
    Modifies existing records in a table. 
    UPDATE students
SET grade = 'B'
WHERE name = 'Alex';
  • 4. Delete data
    Removes records from a table. 
    DELETE FROM students
WHERE name = 'Alex';
  • 5. Filter records (WHERE)
    Retrieves data based on conditions. 
    SELECT * FROM students
WHERE age > 18;
  • 6. Sort records (ORDER BY)
    Sorts results in ascending or descending order. 
    SELECT * FROM students
ORDER BY age ASC;
  • 7. Count records
    Returns the total number of rows. 
    SELECT COUNT(*) AS total_students
FROM students;
  • 8. Average values
    Calculates the average of a numeric column. 
    SELECT AVG(age) AS average_age
FROM students;
  • 9. Group data
    Groups records and applies aggregate functions. 
    SELECT grade, COUNT(*) AS total_students
FROM students
GROUP BY grade;

    JavaScript & TypeScript Notes (Smart Comparison)

    JavaScript (JS) and TypeScript (TS) are closely related. TypeScript is a superset of JavaScript, meaning:
    All JavaScript code is valid TypeScript code.

    • Main Difference
      • JavaScript ‚Üí Dynamic typing
      • TypeScript ‚Üí Static typing + JavaScript features
    • Basic Variable Declaration 
      JavaScript:
let age = 25;

TypeScript:
let age: number = 25;
    • Data Types 
      JavaScript:
let name = "Alex";
let isActive = true;

TypeScript:
let name: string = "Alex";
let isActive: boolean = true;
    • Functions 
      JavaScript:
function add(a, b) {
    return a + b;
}

TypeScript:
function add(a: number, b: number): number {
    return a + b;
}
    • Arrays 
      JavaScript:
let numbers = [1, 2, 3];

TypeScript:
let numbers: number[] = [1, 2, 3];
    • Objects 
      JavaScript:
let user = {
    name: "Sam",
    age: 22
};

TypeScript:
let user: {
    name: string;
    age: number;
} = {
    name: "Sam",
    age: 22
};
    • Why TypeScript Exists
      • Catches errors at compile time
      • Better code readability
      • Great for large applications
      • Excellent IDE support
    • Compilation Flow 
      TypeScript (.ts)
      ‚Üì
Compiled to JavaScript (.js)
      ‚Üì
Runs in Browser / Node.js
    • Where They Are Used
      • JavaScript ‚Üí Browsers, Node.js, web apps
      • TypeScript ‚Üí Angular, React, enterprise apps

    Learning Strategy: Learn JavaScript fundamentals first, then move to TypeScript for safer and scalable code.

    Network Security Notes

    • Network Security refers to protecting networks, devices, and data from unauthorized access, attacks, and misuse.
    • Goals of Network Security (CIA Triad)
      • Confidentiality ‚Äì Data is accessible only to authorized users
      • Integrity ‚Äì Data is not altered without permission
      • Availability ‚Äì Systems and data are accessible when needed
    • Common Network Threats
      • Malware ‚Äì Viruses, worms, trojans
      • Phishing ‚Äì Fake messages to steal credentials
      • Man-in-the-Middle (MITM) ‚Äì Intercepting communication
      • Denial of Service (DoS / DDoS) ‚Äì Overloading a system
      • Packet Sniffing ‚Äì Capturing network traffic
    • Firewalls
      • Filter incoming and outgoing network traffic
      • Types:
        • Packet-filtering firewall
        • Stateful firewall
        • Application-level firewall
        • Next-Generation Firewall (NGFW)
    • Encryption
      • Protects data during transmission
      • Symmetric Encryption ‚Äì Same key for encryption & decryption
      • Asymmetric Encryption ‚Äì Public key & private key
      • Examples: 
        HTTPS ‚Üí TLS/SSL
SSH   ‚Üí Secure remote access
    • Authentication Methods
      • Password-based authentication
      • Multi-Factor Authentication (MFA)
      • Biometric authentication
      • Certificate-based authentication
    • IDS & IPS
      • IDS (Intrusion Detection System) ‚Äì Detects attacks
      • IPS (Intrusion Prevention System) ‚Äì Detects and blocks attacks
    • Secure Network Practices
      • Use strong passwords and MFA
      • Keep systems and software updated
      • Segment networks (VLANs)
      • Disable unused ports and services
      • Monitor logs and traffic
    • Common Secure Ports 
      HTTPS : 443
SSH   : 22
FTPS  : 990
SFTP  : 22

    Cloud Computing Notes

    • 1. Introduction to Cloud Computing
      • Definition: Delivery of services (storage, servers, software etc.) over the Internet.
      • Goal: Use IT resources without owning physical hardware.
      • Example: Storing files on Google Drive instead of a personal PC.
    • 2. Characteristics of Cloud Computing
      • On-demand self-service: Access resources instantly.
      • Broad network access: Accessible anywhere with internet.
      • Resource pooling: Shared infrastructure among users.
      • Rapid elasticity: Scale resources up/down quickly.
      • Measured service: Pay for what you use.
    • 3. Cloud Service Models
      • IaaS: Infrastructure as a Service (e.g., AWS EC2)
      • PaaS: Platform as a Service (e.g., Google App Engine)
      • SaaS: Software as a Service (e.g., Gmail, Zoom)
    • 4. Deployment Models
      • Public Cloud: Services for everyone (e.g., AWS, Azure)
      • Private Cloud: Used by one organization
      • Hybrid Cloud: Public + Private combined
    • 5. Advantages
      • Cost-effective
      • Scalable
      • Accessible from anywhere
      • Automatic updates
      • Backup & disaster recovery
    • 6. Disadvantages
      • Needs internet connection
      • Data security concerns
      • Limited control
      • Possible downtime
    • 7. Real World Applications
      • Storage: Google Drive, Dropbox
      • Streaming: Netflix, Hotstar
      • E-commerce: Amazon
      • Education: Coursera
      • Banking: Secure cloud-based transactions
    • Quick Recap (Easy Examples)
      • IaaS ‚Üí Renting a house (AWS EC2)
      • PaaS ‚Üí Furnished apartment (Google App Engine)
      • SaaS ‚Üí Hotel stay (Gmail, Zoom)
